## 내일: 잔여 시도 토지 엑셀 적재 (출근 전 폴더 준비 후)

**이미 DB·웹에서 다룬 시도(참고)**  
충청북도(43)·경기도(41)·충청남도(44)·경상북도(47) — 이외 시도를 같은 방식으로 순차 확장한다.

**사용자가 출근 전까지 할 일**

1. `원본`(또는 동일 상위 폴더) 아래 **`토지_<시도별>`** 폴더를 만든다.
2. 그 폴더 **바로 안**에만 해당 시도 연도별 국토부 토지 실거래 `.xlsx`를 둔다(하위 폴더 검색 없음). 파일명은 기존처럼 시도명이 들어가 있는 원본 형식이면 된다.
3. 레포의 `data/region_codes/*.csv`(법정동 마스터)·`data/population/*.csv`(인구)는 그대로 두면 된다.

**에이전트/로컬에서 돌릴 파이프라인(시도마다 반복)**

1. `cd pipeline` 후 `DATABASE_URL` 확인(`pipeline/.env`).
2. **`python seed_region_codes.py --file ../data/region_codes/<마스터.csv> --sido "<시도한글명>"`**  
   이름은 `pipeline/seed_region_codes.py` 의 `SIDO_CODE_MAP` 과 정확히 일치해야 한다(예: `강원특별자치도`, `전북특별자치도`).
3. **`python run_pipeline.py --excel-dir "<원본/토지_<시도> 절대경로>" --excel-format auto`**  
   (충청남도·경북과 같이 시간이 걸린다.)
4. **인구**(연도별 표 「연말 인구」): 같은 인구 CSV에 대해  
   **`python seed_population_csv.py --file ../data/population/<파일.csv> --codes-prefix <시도코드2자리>`**  
   시도별 앞두 자리는 README 인구 절 또는 `SIDO_CODE_MAP` 참고.

**참고 스크립트**

- 경북 일괄 예: `pipeline/_launch_gyeongbuk_pipeline.py` (시드 + `run_pipeline` + 인구 47 접두 반복을 한 번에).
- 새 시도는 동일 패턴으로 `--sido`·폴더 검색 문자열(`경상북도` 등)·`--codes-prefix` 만 바꾼 런처를 만들거나, 위 2~4를 수동으로 실행.

**실행 타이밍**

- Cursor 에이전트는 대화나 작업 요청 시에만 돌아간다. 출근 중 자동 실행은 **Windows 작업 스케줄러** 등으로 배치 스크립트를 거는 방식이 필요하다.

## 확인한 핵심 판단

기존 차이는 단순 UI 문제가 아니었다. 주요 원인은 다음이었다.

- 과거 해시가 물건 단위에 가까워 정상 신고와 해제 신고가 서로 덮어쓸 수 있었다.
- 동일 물건에 실제 소유권이전이 있었고, 그중 일부 신고만 해제된 사례가 있었다.
- 따라서 물건 단위로 전체 제거하면 유효 거래까지 빠질 수 있다.

최종 정책은 **신고 행 보존 + 해제 행만 제외**이다.
